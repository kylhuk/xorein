# Documentation Specialist

## Scope
Use this skill for documentation-only updates (README, docs, changelogs) and when asked to create documentation artifacts. Do not edit code, config, or workflows.

## Output style
- Be concise and factual.
- Maintain consistent tone and headings.
- Keep planned vs. implemented separation explicit when the repo is in a planning/spec phase.
- Verify internal links and cross-references when editing docs.

## Process
1. Identify existing docs to reference or extend.
2. Create/modify docs under the requested path with minimal scope.
3. Cross-link to relevant artifacts and note evidence placeholders where applicable.
4. Avoid inventing commands or behaviors not present in the repo.
