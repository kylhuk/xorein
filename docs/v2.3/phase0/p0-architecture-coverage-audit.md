# Architecture coverage audit (Phase 0 P0-T2 ST1–ST3)

This planning artifact inventories **every persisted data class** referenced in TODO_v23 P0-T2 ST1. Each row maps the class to authoritative writers, storage planes, retrieval paths for onboarding devices, confidentiality/integrity guarantees, and retention policy owners/defaults. Unknowns are explicitly flagged for `F24` seeds or future hardening work so `G11` can never hide persistence gaps.

### Missing or unknown persistence classes
- **Relayed cache metadata (e.g., temporary search hints).** If this metadata is ever persisted beyond in-memory caches, it must become an `F24` seed (e.g., `F24-A` for encrypted caches). For now, we treat it as non-persisted. (No evidence recorded.)
- **Assisted search hints for future opt-in workflows.** Explicitly deferred to `F24-C`. Any persistence introduced there must reference `G11`. |

If additional data classes are discovered (e.g., telemetry dumps, diagnostics artifacts, or operator-only caches), append them here and tie them either to `v23` hardening work or to `F24` seed candidates. The final `Phase 4` audit artifact must cite this list plus its approvals so `G11` can close with no unknown persistence.
